# Document Analysis System - Backend API Specification

## Overview

This document provides the complete API specification for the Document Analysis System backend. The system processes financial documents (ELN, Bond, IF) through a multi-step workflow: file upload verification, field extraction, and final confirmation.

**Base URL:** `https://api.yourdomain.com/v1`

**Authentication:** Bearer Token (JWT)
```
Authorization: Bearer <token>
```

---

## Workflow Steps

1. **Upload & Verify** - User uploads files and selects document type
2. **File Type Verification** - Backend verifies file types and formats
3. **Field Extraction** - Backend extracts structured data from documents
4. **User Confirmation** - User reviews and edits extracted data
5. **Final Submission** - Confirmed data is submitted for processing

---

## API Endpoints

### 1. Verify File Types

**Endpoint:** `POST /api/documents/verify`

**Description:** Verifies uploaded files to check if they match the expected document type and format.

**Request Headers:**
```http
Content-Type: multipart/form-data
Authorization: Bearer <token>
```

**Request Body:**
```json
{
  "documentType": "ELN",
  "files": [
    {
      "fileName": "document1.pdf",
      "fileSize": 1024567,
      "fileType": "application/pdf",
      "content": "<binary_data>"
    },
    {
      "fileName": "document2.pdf",
      "fileSize": 2048901,
      "fileType": "application/pdf",
      "content": "<binary_data>"
    }
  ]
}
```

**Parameters:**
- `documentType` (string, required): Type of document - `"ELN"`, `"BOND"`, or `"IF"`
- `files` (array, required): Array of file objects
  - `fileName` (string): Original filename
  - `fileSize` (integer): File size in bytes
  - `fileType` (string): MIME type
  - `content` (binary): File binary content

**Success Response (200):**
```json
{
  "success": true,
  "data": [
    {
      "fileName": "document1.pdf",
      "fileTypeCorrect": true,
      "detectedType": "ELN",
      "message": "File type verified as ELN",
      "confidence": 0.95,
      "extractedMetadata": {
        "pages": 2,
        "wordCount": 450,
        "hasTables": true
      }
    },
    {
      "fileName": "document2.pdf",
      "fileTypeCorrect": false,
      "detectedType": "BOND",
      "message": "Type mismatch: Expected ELN, detected BOND",
      "confidence": 0.88,
      "extractedMetadata": {
        "pages": 1,
        "wordCount": 320,
        "hasTables": false
      }
    }
  ],
  "timestamp": "2024-02-11T08:30:45.123Z",
  "requestId": "req_abc123xyz"
}
```

**Error Response (400):**
```json
{
  "success": false,
  "error": {
    "code": "INVALID_FILE_TYPE",
    "message": "One or more files have unsupported formats",
    "details": [
      {
        "fileName": "document3.exe",
        "reason": "Unsupported file extension"
      }
    ]
  },
  "timestamp": "2024-02-11T08:30:45.123Z",
  "requestId": "req_abc123xyz"
}
```

### 3. Confirm and Submit

**Endpoint:** `POST /api/documents/confirm`

**Description:** Submits the final confirmed data for processing and storage.

**Request Headers:**
```http
Content-Type: application/json
Authorization: Bearer <token>
```

**Request Body:**
```json
{
  "submissionId": "sub_123456",
  "documentType": "ELN",
  "files": [
    {
      "fileName": "document1.pdf",
      "fileType": "ELN",
      "fields": {
        "TRADE_DATE": "2024-01-15",
        "MTR_DATE": "2024-07-15",
        "ISSUER": "Hang Seng Bank",
        "UNDERLYING": "HSI Index",
        "STRIKE_PRICE": "18500",
        "COUPON_RATE": "8.5%",
        "NOTIONAL_AMOUNT": "1000000",
        "CURRENCY": "HKD"
      },
      "originalFields": {
        "TRADE_DATE": "2024-01-15",
        "MTR_DATE": "2024-07-15",
        "STRIKE_PRICE": "18000"
      },
      "modifiedFields": ["STRIKE_PRICE"]
    }
  ],
  "metadata": {
    "totalFiles": 1,
    "totalFields": 8,
    "modifiedFields": 1,
    "processingTime": 5.32
  }
}
```

**Parameters:**
- `submissionId` (string, required): Unique submission identifier
- `documentType` (string, required): Document type
- `files` (array, required): Array of file objects with confirmed fields
  - `fileName` (string): Filename
  - `fileType` (string): Document type
  - `fields` (object): Confirmed/extracted fields
  - `originalFields` (object, optional): Original extracted values before user edits
  - `modifiedFields` (array, optional): List of field names that were modified
- `metadata` (object): Processing metadata

**Success Response (200):**
```json
{
  "success": true,
```

## Data Models

### Document Type Enum
```typescript
enum DocumentType {
  ELN = "ELN",    // Equity Linked Note
  BOND = "BOND",  // Bond
  IF = "IF"       // Investment Fund
}
```

### File Verification Result
```typescript
interface FileVerificationResult {
  fileName: string;
  fileTypeCorrect: boolean;
  detectedType: DocumentType;
  message: string;
  confidence: number;
  extractedMetadata?: {
    pages?: number;
    wordCount?: number;
    hasTables?: boolean;
  };
}
```

### Extracted Field
```typescript
interface ExtractedField {
  fileName: string;
  fileType: DocumentType;
  fields: Record<string, string>;
  extractionConfidence: number;
  extractionTime: number;
  ocrData: {
    engine: string;
    confidence: number;
  };
}
```

### Submission Data
```typescript
interface SubmissionData {
  submissionId: string;
  documentType: DocumentType;
  files: Array<{
    fileName: string;
    fileType: DocumentType;
    fields: Record<string, string>;
    originalFields?: Record<string, string>;
    modifiedFields?: string[];
  }>;
  metadata: {
    totalFiles: number;
    totalFields: number;
    modifiedFields: number;
    processingTime: number;
  };
}
```

| `INVALID_FILE_TYPE` | 400 | Unsupported file format |

| `FORBIDDEN` | 403 | Insufficient permissions |
| `NOT_FOUND` | 404 | Resource not found |
| `EXTRACTION_FAILED` | 422 | Field extraction failed |
| `PROCESSING_FAILED` | 500 | Internal processing error |
| `DATABASE_ERROR` | 500 | Database operation failed |
| `TIMEOUT` | 504 | Request timeout |

## Performance Requirements

- **File Upload:** Max 10MB per file, max 10 files per request
- **Verification:** < 2 seconds for 3 files
- **Extraction:** < 5 seconds per file
- **Confirmation:** < 3 seconds
- **Concurrent Requests:** Support 100 concurrent users
- **Availability:** 99.9% uptime

---

## Security Requirements

1. **Authentication:** JWT Bearer token required for all endpoints
2. **File Validation:**
   - Whitelist file extensions: `.pdf`, `.doc`, `.docx`, `.txt`
   - Scan for malware/viruses
   - Validate MIME types
3. **Data Encryption:**
   - TLS 1.3 for all API communications
   - Encrypt sensitive fields at rest
4. **Rate Limiting:**
   - 100 requests per minute per user
   - 10 file uploads per minute per user
5. **Audit Logging:** Log all document processing activities

## Frontend Integration Notes

### Mock Implementation
For development and testing, the frontend currently uses mock data. The actual backend implementation should match these data structures.

### Key Points:
1. **Step 2 Response** must include `fileTypeCorrect` boolean for each file
2. **Step 3 Response** must include `fields` object with all extracted data
3. **Step 4 Request** may include modified fields - track changes for audit
4. All timestamps should be ISO 8601 format (UTC)

### Example Frontend Usage:
```javascript
// Step 1: Verify files
const verifyResponse = await fetch('/api/documents/verify', {
```

##  Overview

The Document Analysis System is a web application designed to streamline the processing of financial documents including ELN (Equity Linked Notes), Bonds, and Investment Funds (IF). The system provides an intuitive workflow for uploading, verifying, extracting, and confirming document data.

### Workflow

1. ** Upload** - Select document type and upload files
2. ** Verify** - Automatic file type verification with manual correction
3. ** Extract** - AI-powered field extraction with editable results
4. ** Complete** - Final confirmation and submission

##  Features

### Core Features

- **Multi-Document Support**
  - ELN (Equity Linked Notes)
  - Bonds
  - Investment Funds (IF)

- **File Upload**
  - Drag & drop interface
  - Multiple file upload (PDF, DOC, DOCX, TXT)
  - File validation and preview

- **Verification System**
  - Automatic document type detection
  - Manual correction capability
  - Remove/re-add files
  - Checkbox-based status toggling

- **Field Extraction**
  - Structured data extraction
  - Editable table interface
  - Change tracking with visual indicators
  - Reset to original values

- **Professional UI**
  - Material Design components
  - Responsive layout
  - Progress tracking
  - Success/error feedback

### Technical Features

-  Material UI with custom Hang Seng Bank theme (#008945)
-  Fully responsive design
-  Accessibility support (ARIA labels, keyboard navigation)
-  Optimized performance (React.memo, useMemo, useCallback)

##  Components

Multi-file upload component with drag & drop support.

**Props:**
- `files` (array): List of uploaded files
- `onFilesChange` (function): File change handler
- `selectedType` (string): Selected document type
- `onTypeChange` (function): Type change handler
- `documentTypes` (array): Available document types

### FileTypeVerification
Displays file verification results with editing capabilities.

**Props:**
- `results` (array): Verification results
- `onConfirm` (function): Confirm handler
- `onCancel` (function): Cancel handler

**Features:**
- Checkbox toggling for status
- Remove/restore files
- Status chips (Correct/Error)
- Bulk actions

### FieldExtractionTable
Editable table for extracted document fields.

**Props:**
- `data` (array): Extracted field data
- `onConfirm` (function): Submit handler
- `onCancel` (function): Cancel handler

**Features:**
- Inline text editing
- Change tracking (yellow highlight)
- Reset to original values
- Multi-file column view

### FinalConfirmation
Displays processing completion summary.

**Props:**
- `result` (object): Processing result

- Files processed count
- Total records
- Timestamp

import React from 'react';
import { AlertCircle, RefreshCw } from 'lucide-react';

class ErrorBoundary extends React.Component {
  constructor(props) {
    super(props);
    this.state = { hasError: false, error: null, errorInfo: null };
  }

  static getDerivedStateFromError(error) {
    // Update state so the next render will show the fallback UI
    return { hasError: true };
  }

  componentDidCatch(error, errorInfo) {
    // Log error details
    this.setState({
      error: error,
      errorInfo: errorInfo
    });
    
    // Log to console in development
    console.error('Error caught by ErrorBoundary:', error, errorInfo);
  }

  handleReset = () => {
    this.setState({ hasError: false, error: null, errorInfo: null });
    if (this.props.onReset) {
      this.props.onReset();
    }
  };

  render() {
    if (this.state.hasError) {
      return (
        <div className="min-h-screen flex items-center justify-center bg-gray-100 p-4">
          <div className="bg-white rounded-xl shadow-lg p-8 max-w-lg w-full text-center">
            <div className="inline-flex items-center justify-center w-16 h-16 bg-red-100 rounded-full mb-4">
              <AlertCircle className="w-8 h-8 text-red-600" aria-hidden="true" />
            </div>
            
            <h2 className="text-2xl font-bold text-gray-800 mb-4">
              Something went wrong
            </h2>
            
            <p className="text-gray-600 mb-6">
              An unexpected error occurred. Please try again or contact support if the problem persists.
            </p>

            {process.env.NODE_ENV === 'development' && this.state.error && (

                  <pre className="text-xs text-red-600 mt-2 whitespace-pre-wrap">
                    {this.state.error.toString()}
                    {this.state.errorInfo?.componentStack}
                  </pre>
                </details>
              </div>
            )}

            <button
              onClick={this.handleReset}
              className="flex items-center justify-center space-x-2 w-full px-6 py-3

const VALID_EXTENSIONS = ['.pdf', '.doc', '.docx', '.txt'];

const isValidFile = (file) => {
  return VALID_EXTENSIONS.some(ext => file.name.toLowerCase().endsWith(ext));
};

const formatFileSize = (bytes) => {
  if (bytes === 0) return '0 B';
  const k = 1024;
  const sizes = ['B', 'KB', 'MB', 'GB'];
  const i = Math.floor(Math.log(bytes) / Math.log(k));
  return parseFloat((bytes / Math.pow(k, i)).toFixed(1)) + ' ' + sizes[i];
};

const FileUpload = ({ files, onFilesChange, selectedType, onTypeChange, documentTypes }) => {
  const [isDragging, setIsDragging] = useState(false);
  const fileInputRef = useRef(null);

  const handleDragOver = useCallback((e) => {
    e.preventDefault();
    setIsDragging(true);
  }, []);

  const handleDragLeave = useCallback((e) => {
    e.preventDefault();
    setIsDragging(false);
  }, []);

  const handleDrop = useCallback((e) => {
    e.preventDefault();
    setIsDragging(false);
    
    const droppedFiles = Array.from(e.dataTransfer.files);
    const validFiles = droppedFiles.filter(isValidFile);
    
    if (validFiles.length > 0) {
      onFilesChange(prev => [...prev, ...validFiles]);
    }
  }, [onFilesChange]);

  const handleFileInput = useCallback((e) => {
    const selectedFiles = Array.from(e.target.files);
    onFilesChange(prev => [...prev, ...selectedFiles]);
    e.target.value = '';
  }, [onFilesChange]);

  const removeFile = useCallback((index) => {
    onFilesChange(prev => prev.filter((_, i) => i !== index));
  }, [onFilesChange]);

  const handleClick = () => {
    fileInputRef.current?.click();
  };

  return (
    <Box>
      {/* Document Type Selection */}
      <FormControl fullWidth sx={{ mb: 3 }}>
        <InputLabel id="document-type-label">Document Type</InputLabel>
        <Select
          labelId="document-type-label"
          value={selectedType}
          label="Document Type"
          onChange={onTypeChange}
        >
          {documentTypes.map((type) => (
            <MenuItem key={type.value} value={type.value}>
              {type.label}
            </MenuItem>
          ))}
        </Select>
      </FormControl>

      {/* Upload Area */}
      <Paper
        variant="outlined"
        onClick={handleClick}
        onDragOver={handleDragOver}
        onDragLeave={handleDragLeave}
        onDrop={handleDrop}
        sx={{
          p: 4,
          textAlign: 'center',
          cursor: 'pointer',
          border: '2px dashed',
          borderColor: isDragging ? 'primary.main' : 'grey.300',
          bgcolor: isDragging ? 'primary.50' : 'background.paper',
          transition: 'all 0.3s ease',
          '&:hover': {
            borderColor: 'primary.main',
            bgcolor: 'primary.50',
          },
        }}
      >
        <input
          type="file"
          multiple
          ref={fileInputRef}
          onChange={handleFileInput}
          style={{ display: 'none' }}
          accept={VALID_EXTENSIONS.join(',')}
        />
        
        <CloudUpload 
          sx={{ 
            fontSize: 48, 
            color: 'primary.main',
            mb: 2,
          }} 
        />
        
        <Typography variant="h6" gutterBottom color="text.primary">
          Drop files here or click to upload
        </Typography>
        
        <Typography variant="body2" color="text.secondary">
          Support PDF, Word, and Text files
        </Typography>
        
        <Button
          variant="outlined"
          sx={{ mt: 2 }}
          startIcon={<CloudUpload />}
          onClick={(e) => {
            e.stopPropagation();
            handleClick();
          }}
        >
          Select Files
        </Button>
      </Paper>

      {/* File List */}
      {files.length > 0 && (
        <Box sx={{ mt: 3 }}>

            />
          </Box>
          
          <Paper variant="outlined" sx={{ borderRadius: 2 }}>
            <List sx={{ py: 0 }}>
              {files.map((file, index) => (
                <ListItem

                  divider={index < files.length - 1}
